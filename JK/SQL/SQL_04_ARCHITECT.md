# SQL_04_ARCHITECT

## Database Architecture and Systems

*Target Audience*: Staff Engineer / System Architect.  
*Philosophy*: Focus on "Design choices, Trade-offs, and Systems Theory".

### Internals

- Relational algebra foundations.  
- Query optimizer internals.  
- Storage engine architectures (InnoDB, MyISAM).

### Theory

- Relational model theory.  
- Normal forms and their implications.  
- CAP theorem in distributed databases.

### Scalability

- Sharding and partitioning strategies.  
- Read/write splitting.  
- NoSQL integration with SQL systems.

### Design

- Data warehouse vs OLTP design.  
- Microservices database patterns.  
- API design over databases.

### Patterns

- CQRS with event sourcing.  
- Database refactoring patterns.  
- Polyglot persistence architectures.
